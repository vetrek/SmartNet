<task type="auto">
  <name>Task 2: Add Equatable conformance to NetworkError</name>
  <files>Source/Utils/NetworkError.swift</files>
  <action>
Add Equatable conformance to NetworkError enum. This requires a custom `==` implementation because several cases have associated values that need special handling:

- Cases with `Data?`: Compare data directly
- Cases with `Error`: Compare by localizedDescription (Error itself isn't Equatable)
- Cases with `Decodable`: Compare by String(describing:) since Decodable isn't Equatable
- Cases with `URL?`: Compare URLs directly
- Cases with `TimeInterval?`: Compare directly

Implementation approach:
1. Add `Equatable` to the enum declaration
2. Implement `static func == (lhs: NetworkError, rhs: NetworkError) -> Bool`
3. Use switch on (lhs, rhs) tuple to compare matching cases
4. For Error associated values, compare localizedDescription strings
5. Return false for non-matching case pairs

Do NOT add Sendable - it would require breaking changes since Error isn't Sendable.
  </action>
  <verify>swift build succeeds, swift test passes, NetworkError values can be compared with ==</verify>
  <done>NetworkError conforms to Equatable, all tests pass, equality comparisons work correctly</done>
</task>
